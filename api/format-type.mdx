---
title: FormatType enum
description: Enum for specifying phone number output formats
---

The `FormatType` enum allows you to specify the desired format when formatting phone numbers using the `formatNumber()` extension method.

## Enum Values

### regular

Formats the phone number using the standard Zimbabwean format starting with `0`.

<ResponseField name="format" type="String">
  0XXXXXXXXX (10 digits starting with 0)
</ResponseField>

```dart
final phone = '263771234567'.formatNumber(
  formatType: FormatType.regular,
);
print(phone); // 0771234567

final voip = '+263868812345'.formatNumber(
  formatType: FormatType.regular,
);
print(voip); // 0868812345
```

**Use cases:**
- Displaying numbers to Zimbabwean users
- Local dialing within Zimbabwe
- SMS sending within the country

### countryCode

Formats the phone number with the country code `263` without the plus sign.

<ResponseField name="format" type="String">
  263XXXXXXXXX (12 digits starting with 263)
</ResponseField>

```dart
final phone = '0771234567'.formatNumber(
  formatType: FormatType.countryCode,
);
print(phone); // 263771234567

final landline = '0242123456'.formatNumber(
  formatType: FormatType.countryCode,
);
print(landline); // 263242123456
```

**Use cases:**
- API integrations that require numeric-only format
- Database storage without special characters
- Systems that don't support the plus symbol

### countryCodePlus

Formats the phone number with the country code in international format with the plus sign `+263`.

<ResponseField name="format" type="String">
  +263XXXXXXXXX (13 characters starting with +263)
</ResponseField>

```dart
final phone = '0771234567'.formatNumber(
  formatType: FormatType.countryCodePlus,
);
print(phone); // +263771234567

final netone = '0711234567'.formatNumber(
  formatType: FormatType.countryCodePlus,
);
print(netone); // +263711234567
```

**Use cases:**
- International dialing
- WhatsApp Business API
- SMS gateways requiring E.164 format
- Contacts export
- Mobile app display for international users

## Usage Examples

### Converting Between Formats

```dart
final phone = '0771234567';

// Get all three formats
final regular = phone.formatNumber(formatType: FormatType.regular);
final countryCode = phone.formatNumber(formatType: FormatType.countryCode);
final international = phone.formatNumber(formatType: FormatType.countryCodePlus);

print(regular);        // 0771234567
print(countryCode);    // 263771234567
print(international);  // +263771234567
```

### API Integration Example

```dart
future<void> sendSMS(String recipient, String message) async {
  // Format for SMS gateway (requires international format)
  final formattedNumber = recipient.formatNumber(
    formatType: FormatType.countryCodePlus,
  );
  
  await smsService.send(
    to: formattedNumber,
    message: message,
  );
}

// Usage
await sendSMS('0771234567', 'Hello!'); // Sends to +263771234567
```

### User Input Handling

```dart
String formatForDisplay(String rawNumber) {
  try {
    // Clean user input and format for display
    return rawNumber.formatNumber(
      formatType: FormatType.regular,
      cleanNumber: true, // Handle (077) 123-4567 format
    )!;
  } on LocalRegexException {
    return rawNumber; // Return original if invalid
  }
}

print(formatForDisplay('(077) 123-4567')); // 0771234567
print(formatForDisplay('+263 77 123 4567')); // 0771234567
```

### Database Storage Best Practice

```dart
class Contact {
  final String phoneNumber;
  
  Contact(String rawPhone) : phoneNumber = rawPhone.formatNumber(
    formatType: FormatType.countryCode, // Store without +
  )!;
  
  String get displayNumber => phoneNumber.formatNumber(
    formatType: FormatType.regular,
  )!;
  
  String get internationalNumber => phoneNumber.formatNumber(
    formatType: FormatType.countryCodePlus,
  )!;
}

final contact = Contact('0771234567');
print(contact.phoneNumber);          // 263771234567 (stored)
print(contact.displayNumber);        // 0771234567 (for UI)
print(contact.internationalNumber);  // +263771234567 (for export)
```

<Note>

</Note>
